# 19 — Thread organization

**What to build:** Help users find and manage threads through stable ordering, search, pinning, unread state, and archive/restore.

**Blocked by:** 12 — Workspace and thread navigation with drafts; 18 — Thread naming

**Status:** ready-for-agent

- [ ] Thread order is stable during creation and refresh and supports scoped search.
- [ ] Pinned threads persist above history, can be reordered, and return predictably when unpinned.
- [ ] Completion activity marks background threads unread and selecting them advances persisted read state.
- [ ] Archived threads move to a restorable section without deleting Pi sessions.
