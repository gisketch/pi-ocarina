# 19 — Thread organization

**What to build:** Help users find and manage threads through stable ordering, search, pinning, unread state, and archive/restore.

**Blocked by:** 12 — Workspace and thread navigation with drafts; 18 — Thread naming

**Status:** complete

- [x] Thread order is stable during creation and refresh and supports scoped search.
- [x] Pinned threads persist above history, can be reordered, and return predictably when unpinned.
- [x] Completion activity marks background threads unread and selecting them advances persisted read state.
- [x] Archived threads move to a restorable section without deleting Pi sessions.
